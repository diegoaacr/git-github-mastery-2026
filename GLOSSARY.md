# Glossary

Términos básicos de Git y GitHub explicados de forma simple.

---

## Repository

Lugar donde vive tu proyecto y su historial.

## Commit

Registro de cambios guardado en Git.

## Branch

Línea de trabajo independiente.

## Merge

Proceso de unir cambios de una rama en otra.

## Rebase

Forma de reorganizar el historial de commits.

## Remote

Versión del repositorio que está en un servidor.

## Clone

Copia local de un repositorio remoto.

## Pull

Acción de traer cambios del remoto y aplicarlos.

## Push

Acción de subir tus cambios al remoto.

## Staging Area

Zona donde preparas archivos antes del commit.

## Working Directory

Carpeta donde estás trabajando en tus archivos.

## HEAD

Referencia al commit actual.

## Pull Request

Solicitud para revisar e integrar cambios en GitHub.

## Conflict

Problema que aparece cuando Git no puede unir cambios automáticamente.
