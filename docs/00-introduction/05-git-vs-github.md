# Git vs GitHub

Es muy importante entender la diferencia.

---

## 🎯 Objetivo

No confundir Git con GitHub.

---

## 🧠 Git

Git es una herramienta que instalas en tu computadora.

Sirve para:

* Guardar cambios
* Controlar versiones
* Trabajar con historial

👉 Funciona de forma local.

---

## 🌐 GitHub

GitHub es una plataforma en la nube.

Sirve para:

* Guardar repositorios online
* Compartir código# Why Git matters

Git no es solo una herramienta más. Es algo básico en desarrollo.

---

## 🎯 Objetivo

Entender por qué aprender Git es importante.

---

## 💻 Uso en la industria

Hoy en día, casi todos los equipos usan Git.

* Desarrollo web
* Backend
* Mobile
* DevOps

---

## 🤝 Trabajo en equipo

Git permite que varias personas trabajen en el mismo proyecto sin problemas.

Cada uno puede trabajar en su parte.

---

## 🧠 Control de cambios

Con Git puedes:

* Ver qué cambió
* Volver atrás
* Comparar versiones

---

## 🛡️ Seguridad

Si algo sale mal:

👉 puedes recuperar versiones anteriores

---

## 🚀 Mejores prácticas

Git te obliga a trabajar de forma ordenada:

* commits claros
* ramas
* revisiones

---

## 💡 Idea clave

Aprender Git no es opcional si quieres ser profesional.

---

## ⚠️ Importante

Al inicio puede parecer difícil.

Pero después se vuelve una herramienta diaria.

## 🔍 Diferencia clara

* Git = herramienta
* GitHub = plataforma

---

## 🧩 Ejemplo

Puedes usar Git sin GitHub.

Pero no puedes usar GitHub sin Git.

---

## 💡 Analogía

Git es como Word en tu computadora.
GitHub es como Google Drive.

---

## ⚠️ Error común

Pensar que son lo mismo.

No lo son.
