# SHA-1

Git usa hashes SHA para identificar objetos de forma única.

---

## 🎯 Objetivo

Entender cómo funcionan los hashes SHA en Git.

---

## 🧠 Qué es un SHA

Un SHA es un hash generado a partir del contenido de un objeto.

👉 funciona como identificador único

---

## 🎯 Idea clave

Si el contenido cambia, el hash cambia.

---

## 🧩 Ejemplo

```bash id="sha1"
a1b2c3d4e5f6...
```

👉 hash de un objeto Git

---

## 💡 Qué usa Git

Tradicionalmente Git usa:

```bash id="sha2"
SHA-1
```

---

## ⚠️ Importante

Git NO genera hashes aleatorios.

👉 el hash depende completamente del contenido

---

## 🧩 Ejemplo conceptual

```bash id="sha3"
"hello"
↓
hash A
```

```bash id="sha4"
"hello world"
↓
hash B
```

👉 contenido diferente = hash diferente

---

## 💡 Qué identifica Git con hashes

* blobs
* trees
* commits
* tags

👉 todo objeto Git tiene SHA

---

## 🧠 Por qué es importante

Permite:

* integridad
* detección de cambios
* referencias únicas
* historial seguro

---

## ⚠️ Qué pasa si cambia un commit

Si modificas:

* mensaje
* autor
* contenido

👉 cambia el SHA del commit

---

## 🧩 Efecto en cadena

```bash id="sha5"
commit A
↓
commit B
↓
commit C
```

Si cambia `A`:

👉 cambian también `B` y `C`

---

## 💡 Relación con rebase

Por eso:

```bash id="sha6"
git rebase
```

👉 crea commits nuevos con SHAs nuevos

---

## ⚠️ Importante

Los SHAs ayudan a Git a garantizar integridad del historial.

---

## 💡 SHA corto

Normalmente usamos:

```bash id="sha7"
a1b2c3
```

👉 versión corta del hash completo

---

## ⚠️ Error común

Pensar que el SHA es “solo un número aleatorio”.

👉 realmente representa el contenido exacto

---

## 💡 Buenas prácticas

* usar SHAs cortos para referencia rápida
* entender que cambiar commits cambia hashes

---

## 🧠 Nivel profesional

Comprender SHAs ayuda muchísimo para:

* rebases
* debugging
* recuperación
* internals avanzados
