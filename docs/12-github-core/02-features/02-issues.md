# Issues

Los Issues permiten gestionar tareas, bugs y mejoras en GitHub.

---

## 🎯 Objetivo

Entender cómo usar Issues para organizar trabajo.

---

## 🧠 Qué es un Issue

Un Issue representa algo pendiente en el proyecto.

Puede ser:

* bug
* feature
* mejora
* tarea

---

## 🧩 Ejemplo

```bash id="iss1"
Bug: login no funciona en mobile
```

---

## 🎯 Idea clave

Los Issues ayudan a organizar y discutir trabajo.

---

## 💡 Qué puede incluir un Issue

* título
* descripción
* comentarios
* labels
* assignees
* milestones

---

## 🔄 Flujo típico

```bash id="iss2"
crear issue
↓
discutir
↓
trabajar
↓
PR relacionado
↓
cerrar issue
```

---

## 🧠 Relación con Pull Requests

Un PR puede cerrar un Issue automáticamente:

```bash id="iss3"
Fixes #12
```

👉 al mergear el PR, el Issue se cierra

---

## 💡 Labels comunes

```bash id="iss4"
bug
feature
enhancement
documentation
```

---

## ⚠️ Importante

Los Issues NO son código.

👉 son gestión y comunicación

---

## 💡 Beneficios

* organización
* trazabilidad
* comunicación
* planificación

---

## ⚠️ Errores comunes

* Issues sin contexto
* títulos poco claros
* no actualizar estado
* mezclar varios problemas en uno

---

## 💡 Buenas prácticas

* títulos claros
* descripción detallada
* usar labels
* relacionar PRs e Issues

---

## 🧠 En empresas reales

Los Issues suelen conectarse con:

* sprints
* boards
* automatizaciones
* releases
