# Repositories

Un repository es el espacio donde vive un proyecto en GitHub.

---

## 🎯 Objetivo

Entender cómo funcionan los repositories en GitHub.

---

## 🧠 Qué es un repository

Es un contenedor que guarda:

* código
* historial Git
* branches
* Pull Requests
* issues
* releases

---

## 🧩 Ejemplo

```bash id="repo1"
github.com/user/proyecto
```

👉 representa un repository

---

## 🎯 Idea clave

El repository es el centro del proyecto.

---

## 💡 Qué contiene normalmente

```bash id="repo2"
README.md
src/
docs/
.gitignore
LICENSE
```

---

## 🔄 Tipos de repositorios

### Públicos

```bash id="repo3"
public
```

👉 cualquiera puede verlo

---

### Privados

```bash id="repo4"
private
```

👉 acceso restringido

---

## 💡 Qué puedes hacer

* clonar
* forkear
* colaborar
* abrir PRs
* crear releases

---

## ⚠️ Importante

Cada repository tiene:

* URL propia
* permisos
* configuración
* historial independiente

---

## 🧠 Ejemplo real

```bash id="repo5"
frontend-app
backend-api
mobile-app
```

👉 cada uno puede ser un repository diferente

---

## ⚠️ Errores comunes

* mezclar proyectos distintos
* subir secretos
* no documentar el repo

---

## 💡 Buenas prácticas

* usar README claro
* organizar carpetas
* proteger ramas importantes
* mantener historial limpio

---

## 🧠 En empresas reales

Los repositories suelen incluir:

* CI/CD
* reglas de protección
* templates
* automatizaciones
