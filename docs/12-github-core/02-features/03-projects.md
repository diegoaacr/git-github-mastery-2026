# Projects

GitHub Projects permite organizar trabajo usando tableros y planificación visual.

---

## 🎯 Objetivo

Entender cómo gestionar tareas y flujo de trabajo con Projects.

---

## 🧠 Qué es GitHub Projects

Es una herramienta para organizar Issues y Pull Requests.

👉 similar a Trello o Jira

---

## 🧩 Ejemplo típico

```bash id="proj1"
Todo
In Progress
Done
```

---

## 🎯 Idea clave

Projects ayuda a visualizar el estado del trabajo.

---

## 💡 Qué puedes organizar

* Issues
* Pull Requests
* tareas
* roadmap

---

## 🔄 Flujo típico

```bash id="proj2"
crear issue
↓
agregar a project
↓
mover entre columnas
↓
cerrar tarea
```

---

## 🧠 Vista común

### Kanban Board

```bash id="proj3"
Todo → Doing → Done
```

👉 muy usado en equipos ágiles

---

## 💡 Beneficios

* visibilidad
* organización
* seguimiento
* coordinación del equipo

---

## ⚠️ Importante

Projects NO reemplaza Git.

👉 complementa el flujo de trabajo

---

## 💡 Ejemplo real

```bash id="proj4"
Sprint Frontend
```

Tareas:

* login
* dashboard
* navbar

---

## ⚠️ Errores comunes

* boards desactualizados
* demasiadas columnas
* tareas sin contexto
* no cerrar tareas terminadas

---

## 💡 Buenas prácticas

* mantener boards simples
* actualizar estados
* conectar Issues y PRs
* usar nombres claros

---

## 🧠 En empresas reales

Projects suele integrarse con:

* Issues
* Pull Requests
* milestones
* automatizaciones
