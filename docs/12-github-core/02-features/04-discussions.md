# Discussions

GitHub Discussions permite conversar y compartir ideas dentro del proyecto.

---

## 🎯 Objetivo

Entender para qué sirve GitHub Discussions.

---

## 🧠 Qué es Discussions

Es un espacio de conversación dentro del repository.

👉 separado de Issues y Pull Requests

---

## 🎯 Idea clave

Discussions está pensado para hablar y colaborar, no para gestionar código directamente.

---

## 💡 Para qué se usa

* preguntas
* ideas
* feedback
* anuncios
* soporte

---

## 🧩 Ejemplo

```bash id="disc1"
¿Cómo deberíamos organizar la arquitectura del proyecto?
```

---

## 🔄 Diferencia con Issues

| Discussions  | Issues            |
| ------------ | ----------------- |
| conversación | trabajo pendiente |
| preguntas    | bugs/features     |
| ideas        | tareas            |

---

## ⚠️ Importante

No todo debe ser un Issue.

👉 muchas veces primero se discute

---

## 💡 Beneficios

* centraliza conversaciones
* mejora colaboración
* evita ruido en Issues
* ayuda a comunidades open source

---

## 🧠 Categorías comunes

```bash id="disc2"
Q&A
Ideas
Announcements
General
```

---

## ⚠️ Errores comunes

* usar Issues para preguntas simples
* mezclar bugs con conversaciones
* no organizar categorías

---

## 💡 Buenas prácticas

* usar títulos claros
* separar preguntas y bugs
* mantener discusiones organizadas
* documentar decisiones importantes

---

## 🧠 En proyectos open source

Discussions suele usarse para:

* soporte
* comunidad
* propuestas
* feedback
