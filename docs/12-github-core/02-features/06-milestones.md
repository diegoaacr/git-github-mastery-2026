# Milestones

Los Milestones permiten agrupar Issues y Pull Requests hacia un objetivo común.

---

## 🎯 Objetivo

Entender cómo usar Milestones para planificar entregas y versiones.

---

## 🧠 Qué es un Milestone

Es un objetivo o etapa del proyecto.

👉 normalmente asociado a una versión o sprint

---

## 🧩 Ejemplo

```bash id="mile1"
Milestone:
v1.0 Release
```

Incluye:

* login
* dashboard
* responsive layout

---

## 🎯 Idea clave

Un milestone agrupa trabajo relacionado.

---

## 💡 Para qué sirve

* planificar releases
* organizar sprints
* medir progreso
* agrupar tareas

---

## 🔄 Flujo típico

```bash id="mile2"
crear milestone
↓
asignar issues
↓
trabajar
↓
cerrar tareas
↓
completar milestone
```

---

## 🧠 Qué puede contener

* Issues
* Pull Requests

---

## 💡 Ejemplo real

```bash id="mile3"
Sprint 1
```

Tareas:

* fix navbar
* add auth
* improve UI

---

## ⚠️ Importante

El milestone NO hace merge ni despliegues.

👉 es organización y planificación

---

## 💡 Beneficios

* visibilidad del progreso
* mejor planificación
* coordinación del equipo
* seguimiento de releases

---

## ⚠️ Errores comunes

* milestones demasiado grandes
* tareas sin estimación
* no actualizar progreso

---

## 💡 Buenas prácticas

* definir objetivos claros
* usar fechas límite realistas
* cerrar tareas terminadas
* mantener milestones simples

---

## 🧠 En empresas reales

Milestones suelen representar:

* releases
* sprints
* entregas
* versiones importantes
