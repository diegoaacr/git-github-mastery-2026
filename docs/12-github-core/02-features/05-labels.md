# Labels

Los Labels permiten clasificar y organizar Issues y Pull Requests.

---

## 🎯 Objetivo

Entender cómo usar Labels para organizar trabajo en GitHub.

---

## 🧠 Qué es un Label

Es una etiqueta que ayuda a identificar el tipo o estado de algo.

👉 Issues y PRs pueden tener labels

---

## 🧩 Ejemplo

```bash id="lab1"
bug
feature
documentation
urgent
```

---

## 🎯 Idea clave

Los labels ayudan a encontrar y organizar trabajo rápidamente.

---

## 💡 Para qué sirven

* clasificar bugs
* identificar prioridades
* filtrar trabajo
* organizar equipos

---

## 🔄 Ejemplo real

```bash id="lab2"
Issue:
"Navbar broken on mobile"

Labels:
bug
frontend
high-priority
```

---

## 🧠 Labels comunes

| Label         | Uso                    |
| ------------- | ---------------------- |
| bug           | errores                |
| feature       | nuevas funcionalidades |
| enhancement   | mejoras                |
| documentation | docs                   |
| urgent        | prioridad alta         |

---

## ⚠️ Importante

Los labels NO cambian el código.

👉 solo ayudan a organizar trabajo

---

## 💡 Beneficios

* mejor organización
* búsqueda rápida
* priorización
* mejor comunicación

---

## ⚠️ Errores comunes

* demasiados labels
* nombres inconsistentes
* labels sin uso claro

---

## 💡 Buenas prácticas

* mantener nombres simples
* usar colores consistentes
* definir convenciones del equipo
* evitar duplicados

---

## 🧠 En empresas reales

Los labels suelen representar:

* prioridad
* área técnica
* tipo de tarea
* estado del trabajo

---

## 🧠 Ejemplo profesional

```bash id="lab3"

backend

```
