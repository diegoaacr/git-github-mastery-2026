# What is Collaboration

La colaboración en Git permite que varias personas trabajen en el mismo proyecto.

---

## 🎯 Objetivo

Entender cómo Git y GitHub permiten trabajo en equipo.

---

## 🧠 Qué es colaboración

Es el proceso de:

* compartir cambios
* revisar código
* integrar trabajo
* coordinar equipos

---

## 🧩 Ejemplo simple

```bash id="c1"
Developer A → feature-login
Developer B → feature-payment
```

👉 ambos trabajan al mismo tiempo

---

## 🎯 Idea clave

Git permite trabajo paralelo sin romper el proyecto.

---

## 🧠 Cómo se logra

Usando:

* branches
* remotes
* pull requests
* merge

---

## 🔄 Flujo típico

```bash id="c2"
crear branch
↓
trabajar
↓
push
↓
pull request
↓
review
↓
merge
```

---

## 💡 Qué aporta GitHub

GitHub agrega herramientas para:

* revisar código
* comentar cambios
* gestionar equipos
* automatizar procesos

---

## ⚠️ Importante

Trabajar en equipo requiere:

* comunicación
* orden
* buenas prácticas

---

## 💡 Beneficios

* desarrollo más rápido
* menos errores
* código revisado
* historial claro

---

## ⚠️ Errores comunes

* trabajar directamente en main
* no sincronizar cambios
* hacer commits desordenados

---

## 💡 Buenas prácticas

* usar branches por feature
* hacer pull frecuentemente
* usar pull requests
* revisar código antes de merge
