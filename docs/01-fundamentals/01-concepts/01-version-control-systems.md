# Version Control Systems

Los sistemas de control de versiones permiten guardar y gestionar los cambios de un proyecto.

---

## 🎯 Objetivo

Entender qué es un sistema de control de versiones y por qué es importante.

---

## 🧠 Explicación simple

Un sistema de control de versiones guarda el historial de cambios de un proyecto.

Cada vez que haces un cambio, puedes registrarlo y volver atrás si algo sale mal.

---

## 📦 Qué permite hacer

* Guardar versiones de tu proyecto
* Volver a versiones anteriores
* Ver qué cambió
* Trabajar en equipo sin conflictos

---

## 🧩 Ejemplo simple

Imagina que tienes un archivo:

```text
index.html
```

Haces cambios hoy, mañana y pasado.

Sin control de versiones:

* pierdes versiones anteriores
* no sabes qué cambió

Con control de versiones:

* puedes ver cada cambio
* puedes volver atrás
* puedes comparar versiones

---

## 🧠 Idea clave

Un sistema de control de versiones es como un historial inteligente de tu proyecto.

---

## 📊 Tipos de sistemas

Existen dos tipos principales:

### Centralizados

* Un solo servidor
* Todos dependen de ese servidor

### Distribuidos

* Cada persona tiene una copia completa
* No dependes de un servidor central

👉 Git pertenece a este tipo.

---

## ⚠️ Problemas sin control de versiones

* Archivos duplicados (`final_v2_final_ok.js`)
* Pérdida de cambios
* Dificultad para trabajar en equipo

---

## 💡 Por qué es importante

Hoy en día, cualquier proyecto serio usa control de versiones.
