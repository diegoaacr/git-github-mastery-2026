# Versioning Strategies

Versionar correctamente es clave para trabajar en proyectos reales.

---

## 🎯 Objetivo

Entender cómo nombrar versiones de forma profesional.

---

## 🧠 Qué es versionar

Es asignar un número o etiqueta a una versión del proyecto.

👉 normalmente usando tags

---

## 🧩 Ejemplo

```bash
v1.0.0
v1.1.0
v2.0.0
```

---

## 🧠 Estrategia más usada: Semantic Versioning

Formato:

```bash
MAJOR.MINOR.PATCH
```

---

## 🔍 Qué significa

### 1️⃣ MAJOR

```bash
v2.0.0
```

👉 cambios grandes o incompatibles

---

### 2️⃣ MINOR

```bash
v1.1.0
```

👉 nuevas funcionalidades compatibles

---

### 3️⃣ PATCH

```bash
v1.0.1
```

👉 correcciones de errores

---

## 🎯 Idea clave

Cada número tiene un significado claro.

---

## 💡 Ejemplo real

```bash
v1.0.0 → primera versión estable  
v1.0.1 → bug fix  
v1.1.0 → nueva feature  
v2.0.0 → cambio grande  
```

---

## ⚠️ Importante

Mantener consistencia en el versionado.

👉 no cambiar reglas a mitad del proyecto

---

## 💡 Buenas prácticas

* usar formato `vX.Y.Z`
* documentar cambios
* usar tags para versiones

---

## ⚠️ Errores comunes

* usar versiones sin criterio
* saltar números sin sentido
* no diferenciar tipos de cambios
