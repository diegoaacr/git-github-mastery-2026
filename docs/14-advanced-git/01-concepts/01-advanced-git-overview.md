# Advanced Git Overview

Git tiene herramientas avanzadas para resolver problemas complejos y trabajar de forma más profesional.

---

## 🎯 Objetivo

Entender qué tipo de herramientas avanzadas existen en Git.

---

## 🧠 Qué significa “Advanced Git”

Son funcionalidades más poderosas y especializadas.

👉 normalmente usadas en proyectos reales y equipos grandes

---

## 🎯 Idea clave

Estas herramientas ayudan a:

* recuperar cambios
* investigar bugs
* automatizar procesos
* trabajar con repositorios complejos

---

## 💡 Herramientas que aprenderás

### Cherry-pick

```bash id="adv1"
git cherry-pick
```

👉 copiar commits específicos

---

### Bisect

```bash id="adv2"
git bisect
```

👉 encontrar commits problemáticos

---

### Hooks

```bash id="adv3"
git hooks
```

👉 automatizar tareas

---

### Clean

```bash id="adv4"
git clean
```

👉 limpiar archivos no rastreados

---

### Submodules

```bash id="adv5"
git submodule
```

👉 repositorios dentro de otros repositorios

---

### Worktree

```bash id="adv6"
git worktree
```

👉 múltiples ramas activas al mismo tiempo

---

### Blame

```bash id="adv7"
git blame
```

👉 ver quién modificó cada línea

---

## ⚠️ Importante

Estas herramientas son poderosas.

👉 algunas pueden modificar historial o borrar datos

---

## 💡 Cuándo se usan

* debugging
* monorepos
* CI/CD
* equipos grandes
* automatización

---

## ⚠️ Errores comunes

* usar herramientas avanzadas sin entenderlas
* modificar historial incorrectamente
* automatizar procesos peligrosos

---

## 💡 Buenas prácticas

* practicar en repositorios de prueba
* entender el impacto antes de ejecutar comandos
* usar documentación oficial

---

## 🧠 Nivel profesional

Muchos developers usan Git básico.

👉 dominar estas herramientas te diferencia mucho
