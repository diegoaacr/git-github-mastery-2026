# What is a Workflow

Un workflow define cómo un equipo organiza y gestiona cambios en Git.

---

## 🎯 Objetivo

Entender qué es un workflow y por qué es importante.

---

## 🧠 Qué es un Workflow

Es un conjunto de reglas y procesos para trabajar con Git.

👉 define cómo colaborar de forma ordenada

---

## 🎯 Idea clave

Git permite muchas formas de trabajo.

👉 el workflow define cuál usar

---

## 🧩 Ejemplo simple

```bash id="wf1"
crear branch
↓
trabajar
↓
Pull Request
↓
review
↓
merge
```

---

## 💡 Qué controla un workflow

* uso de ramas
* merges
* releases
* reviews
* despliegues

---

## ⚠️ Importante

Un mal workflow genera:

* conflictos
* caos
* deploys inseguros
* historial desordenado

---

## 💡 Objetivos de un buen workflow

* colaboración clara
* estabilidad
* velocidad
* control de cambios

---

## 🧠 Workflows comunes

### GitHub Flow

```bash id="wf2"
simple y moderno
```

---

### Git Flow

```bash id="wf3"
más estructurado
```

---

### Trunk-Based Development

```bash id="wf4"
integración rápida y frecuente
```

---

## 💡 Cómo elegir workflow

Depende de:

* tamaño del equipo
* frecuencia de releases
* complejidad del proyecto
* cultura del equipo

---

## ⚠️ Errores comunes

* usar workflows demasiado complejos
* no documentar procesos
* cambiar reglas constantemente

---

## 💡 Buenas prácticas

* mantener workflows claros
* automatizar donde sea posible
* usar PRs y reviews
* proteger ramas importantes

---

## 🧠 En empresas reales

Los workflows suelen combinarse con:

* CI/CD
* branch protection
* testing automático
* releases controlados
