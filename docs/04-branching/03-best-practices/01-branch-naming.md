01-branch-naming.md# Branch Naming

Nombrar correctamente las ramas es clave para trabajar en equipo.

---

## 🎯 Objetivo

Aprender a usar nombres claros y profesionales para las ramas.

---

## 🧠 Por qué importa el nombre

El nombre de una rama debe explicar:

👉 qué cambio contiene
👉 qué problema resuelve

---

## ❌ Malos ejemplos

```bash id="bn1"
test
cambio
nuevo
rama1
```

👉 No dicen nada.

---

## ✅ Buenos ejemplos

```bash id="bn2"
feature-login
bugfix-header
hotfix-payment-error
```

👉 Son claros y descriptivos.

---

## 🧩 Estructura común

```bash id="bn3"
tipo-descripcion
```

Ejemplos:

```bash id="bn4"
feature-user-auth
bugfix-navbar-alignment
hotfix-critical-error
```

---

## 🔄 Tipos más usados

```bash id="bn5"
feature/
bugfix/
hotfix/
refactor/
```

---

## 🧠 Ejemplo real

```bash id="bn6"
feature/add-login
bugfix/fix-navbar
hotfix/payment-error
```

---

## 🎯 Idea clave

El nombre de la rama debe ser claro y específico.

---

## ⚠️ Importante

Evita nombres genéricos.

👉 dificultan el trabajo en equipo

---

## 💡 Buenas prácticas

* usar inglés
* usar nombres descriptivos
* usar prefijos (`feature/`, `bugfix/`)
* mantener consistencia

---

## ⚠️ Errores comunes

* nombres ambiguos
* mezclar idiomas
* no seguir un estándar

---

Ahora que sabes nombrar ramas:

👉 aprenderás a visualizar cómo se relacionan
